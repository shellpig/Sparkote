# Sparkote 遊戲規格書

> 本文件記錄全遊戲通用系統的規格與驗收條件,是實作時的主要依據之一。
>
> - 早期發想見 `Sparkote_設計文件.md`——那是發展期筆記,僅供參考,**不是實作依據**。
> - 內容 / 場景專屬規格(各地圖、事件劇情、家園佈置細節)之後放 `subdocs/`,由本書 link 過去,不在此展開。
> - 每個系統段含「**驗收**」:可逐條 check 的**意圖層**條件(什麼狀態必須為真),用於手動驗收與已知問題追蹤。**不寫操作步驟**——按哪顆鍵、看什麼畫面的逐步清單在 `測試指南.md`。
> - 撰寫紀律(沿用 AfterTheModel):單一事實來源(同一件事只寫一處,其他 link);未定的標「待定」不杜撰;內容與機制分離。

## 0. 指導原則

- **架構以完整版設計;薄原型只薄在表現層(畫面/UI/美術)。** 系統邏輯、資料模型、參數、串接方式從第一天就是完整版形狀;日後是換/長表現層,不重寫系統。
- **薄原型模式(佔位畫面 + 完整架構 + 真內容 + 測試)橫跨 Phase 1+2**,是第一版全部內容的生產方式;美術延到 Phase 4。
- **系統/資料層蓋一次(永久);表現層由薄變厚(可替換)。** 每個 phase 不是在蓋系統,就是在加厚表現層,兩者分離。

## 2. 核心循環

```
現實完成小任務  →  獲得能量(Sparkote)  →  消耗能量翻開地圖未知格
        ↑                                              ↓
   每日回來推進  ←  資源回饋家園 / 推進劇情  ←  開格獲得資源・物品・事件
```

整個體驗壓在這條轉換鏈的「手感」上:**能量的取得速度 vs 消耗速度的平衡**是本作成敗關鍵,須實機反覆調整(見 Phase 2 的 D1–D7 手感閘門)。

核心信念:**不讓角色變強,而是讓世界變完整。** 回饋來自「未知世界被打開」「家園被修好」「做過的事被日記留下」,不是數值變高。

## 3. 系統分層(總覽)

| 層 | 內容 | 性質 |
|---|---|---|
| ① 狀態脊椎 | `GameState`(玩家狀態單一事實來源) | 蓋一次,永久 |
| ② 邏輯系統 | TaskSystem / Exploration / Requirements / EventSystem / HomeSystem / ResourcePoint / Store / Notification / DayCycle | 蓋一次,永久 |
| ③ 跨切面基礎建設 | Tuning(全域設定)/ Content(內容載入)/ SaveService / AdService / UINavigation | 蓋一次,永久 |
| ④ 表現層 | 各系統的面板(任務/地圖/家園/背包/日記/商店/事件播放) | 薄→厚,可替換 |

**邊界鐵律:**
- 「能量」不是獨立系統,是 `GameState` 的一個欄位 + 全域規則 + 各系統呼叫的 produce/consume。
- 「背包」不是獨立狀態,是 `GameState` 既有資料的**檢視層**。
- `GameState` 只做**原子變更 + 查詢 + requirements 判定**;**編排流程**(翻一格要做的一連串事)住在邏輯系統,由它們呼叫 `GameState` 原子 API + 讀 Content。

## 4. GameState(狀態脊椎)

全域玩家狀態 Autoload,是單一事實來源;存檔即把它序列化。

### 4.1 職責(管理)

遊戲日 / 週進度・能量(含跨日結轉)・資源持有量・任務道具(含關鍵物)・已解鎖地圖 / 已翻開格 ・旗標(系統解鎖 / 劇情進度)・已完成事件・家園狀態(修復 / 插槽擺放)・擁有物(家具 / 外觀 / 收藏)・購買權益・超級任務次數・當日任務狀態・常用任務・日記。

### 4.2 不負責

UI 繪製・場景 / 頁面切換・事件圖文播放・廣告 SDK・字串格式化 / log・**編排流程**(flip / complete_task 等在邏輯系統)。

### 4.3 狀態欄位

| 群組 | 欄位 | 說明 |
|---|---|---|
| 時間 / 進度 | 遊戲日、本週全完成天數、超級任務次數 | 週滿天數換超級任務次數 |
| 能量 | 能量值 | 跨日保留;上限/門檻讀全域設定 |
| 資源 | 資源持有(id→數量) | 基礎建材 + 普通特殊材 + 世界特產 |
| 任務道具 | 道具持有(id→數量) | 含關鍵物,類別由 Content metadata 標 |
| 探索 | 已解鎖地圖、已翻開格(map→tiles)、資源點狀態(採集記錄) | 阻擋解除**不存**,即時判定 |
| 旗標 / 事件 | 旗標(命名空間 key)、已完成事件(event→{日,選項}) | 已完成事件三用:防重發獎 / 回看 / 記選項 |
| 家園 | 修復狀態(repair→等級)、插槽擺放(slot→家具) | |
| 擁有物 | 擁有家具、擁有外觀、已套用外觀、收藏 | 背包 view 顯示用 |
| 變現 | 購買權益(product→true)、每日免費領取旗標、每日資源補給旗標、廣告額外任務旗標 | 後三者 advance_day 重置 |
| 當日任務 | 普通/高級候選、已選普通/高級、廣告額外高級、已完成、超級任務已做 | **進存檔**,跨日才重置 |
| 常用任務 | 常用普通(≤6)、常用高級(≤3) | 跨日持久,替代候選 |
| 日記 | 日→{完成任務,獲得能量,翻格數,解鎖事件,家園變化,心情筆記} | |

### 4.4 存檔

整包狀態可序列化(to_dict)與還原(from_dict),由 `SaveService` 落地。還原後狀態須與原本完全一致(含當日任務、日記、旗標)。

### 4.5 驗收

- 新遊戲後初始狀態正確;每個原子操作正確改值並發對應 signal。
- 序列化→還原後狀態與原本完全一致。
- 能量原子操作:加值在上限內 clamp、達上限發溢出訊號;支付不足時回失敗且不扣值;能量不會變負、不會超上限。
- `requirements` 判定:對給定狀態 + 條件回正確結果;能列出未滿足項。
- 所有狀態變更經 `GameState` API;邏輯系統不繞過它直接改欄位。

## 5. 全域設定(固定常數 + 少數全域值)

跟內容無關、集中一處的數字。**注意:逐格翻開成本不在這裡——成本由各格自帶(見 §8 / §15)。**

| 設定 | 暫定值 | 說明 |
|---|---|---|
| 任務能量 | 普通 1 / 高級 2 / 超級 3 | 固定常數 |
| 能量上限 | 20 | 超過不累積,給明確提示 |
| 輕提示門檻 | 10 | 持有超過開始輕提示 |
| 近上限門檻 | 18 | 提示更明確 |
| 每日普通槽位 / 候選 | 2 / 6 | 6 選 2 |
| 每日高級槽位 / 候選 | 1 / 3 | 3 選 1 |
| 資源點重採成本 | 1 能量 | |
| 資源點每日採集次數 | 1~2(待定) | |
| 週超級任務門檻 | 4 天全完成 | |

### 驗收
- 上述值集中單一來源、可讀;系統不在他處散落同義魔術數字。

## 6. 任務系統(TaskSystem)

玩家在現實完成的小事,是驅動全遊戲的唯一能量來源;完成採**自我申報**,不做拍照 / 定位 / 感測器強驗證。

### 規則
- **每日候選**:每日先產生候選清單(普通 6、高級 3),玩家選 2 普通 + 1 高級為今日任務。
- **常用任務**:玩家可儲存常用任務(普通 ≤6、高級 ≤3),**替代**對應候選位;常用任務只替代候選,**不增加**每日可完成數。
- **完成 → 固定能量**:完成已選且未完成的任務,給該類別固定能量(§5)。
- **看廣告額外高級任務**:每日 1 次,從當日未選高級候選中再選 1 個;免費玩家經 rewarded ad(Phase 1 為 stub),無廣告玩家點擊直接解鎖。額外任務不另外生成新任務。
- **超級任務**:不是每日固定任務,而是玩家獲得的特殊次數;由「一週 4 天全完成」或劇情 / 地圖事件發放。內容玩家自訂,系統只發次數;完成 +3 能量;每日最多 1 個;次數可保留(上限待定)。
- **防作弊**:靠每日槽位、候選清單與能量上限控制,不做強驗證。
- **新手贈點**:**系統不帶任何 gift 邏輯。** D1 的額外贈點(設計文件暫定 +6)完全靠 Content 把早期事件的 rewards 設計出來達成,不在 TaskSystem 內做「首次完成 +1」這類特例。

### 驗收
- 每日產生候選(6 普 / 3 高);常用任務替代對應候選位;常用任務不增加可完成數。
- 選擇受槽位上限;不能超選;完成前可改選。
- 完成「已選且未完成」者給正確固定能量一次;不能完成未選 / 已完成 / 超過槽位者。
- 看廣告額外高級:成功後從未選高級候選解 1 個,每日一次。
- 超級任務:有次數才能消耗,完成 +3 能量,每日最多 1。
- 完成任務除固定能量外不另發新手贈點(贈點來自事件)。

## 7. 能量

能量是 `GameState` 欄位 + 全域規則 + 各系統呼叫的 produce/consume,不是獨立系統。

### 規則
- **產出**:任務完成、超級任務、看廣告額外任務、事件 rewards。
- **消耗**:翻開地圖格(逐格成本)、資源點重採。
- **上限 20、跨日保留**;超過上限不累積,給明確提示。
- **輕提示**:持有 >10 開始輕提示去探索;近上限(18+)提示更明確;**語氣溫和,提醒可探索,不製造壓力**。

### 驗收
- 能量永不超過上限、不會變負;超出捨棄並給明確提示;跨日保留。
- 各產出 / 消耗來源都經 `GameState` 能量 API。
- 過門檻時觸發對應輕提示。

## 8. 探索 / 翻格(Exploration)

地圖以「走格子、打開未知格」推進。

### 規則
- 一張地圖是一個小場景;未探索格被霧遮蔽。
- **相鄰是一張 graph**(每格明列鄰居),與視覺等角座標分離;只能翻開「鄰接已開格的未知格」。
- **逐格成本**:每格自帶翻開成本(一般約 1、特殊 2~3),成本是格子內容,不是全域分類。
- **岔路**:同時多個相鄰選項、玩家取捨;**不做永久錯過**,可回頭補探索。
- 翻開流程(編排):檢查可翻 → 扣該格成本 → 標記翻開 → 依地點類型結算內容。
- **出口格**也需消耗能量翻開,解鎖後可開 1 張或多張新地圖;玩家可在已解鎖地圖間切換。
- **解鎖新地圖時,自動翻開該圖的單一起點格**(比照第一張圖),確保新圖有可探索邊界、不會是動不了的死圖。第一版每張圖單一起點,不做多入口(同一張圖被不同出口接到不同入口的情形延後)。

### 驗收
- 起點預開、其餘遮霧;只有鄰接已開格的未知格可翻,非鄰接者不提供。
- 可翻判定三關:相鄰 + requirements 滿足 + 能量足夠;擋下時回正確原因。
- 翻開精準扣該格自身成本、標記翻開、依類型正確結算。
- 岔路:多選項並存可擇一;略過的格之後仍可補翻。
- 出口格翻開後解鎖指定新圖,且新圖的單一起點格自動翻開、有可翻邊界;可檢視 / 切換已解鎖地圖。

## 9. 地點 + Requirements

### 地點內容類型
| 類型 | 說明 |
|---|---|
| 普通發現 | 解鎖後取得資源 / 小物 / 簡短文字 |
| 資源點 | 初次解鎖給資源,之後可投入能量重複採集 |
| 事件地點 | 插圖 / 對話 / 劇情,記入日記 |
| 裝飾獎勵 | 解鎖家具 / 外觀 / 佈置物 |
| 出口地點 | 解鎖後開啟 1 張或多張新地圖 |

### Requirements(通用阻擋)
- 阻擋是**地點屬性**,任何類型的地點都可被擋。
- 沒有 requirements 的格,可直接花能量翻;有且未滿足者不能花能量,顯示阻擋。
- requirements 滿足後變可翻,翻時仍需花能量。
- **阻擋解除狀態不存**,每次以當前 `GameState` 即時判定(無狀態漂移、存檔更小)。
- 第一版條件類型:持有任務道具 / 完成指定事件 / 解鎖指定地圖 / 達成指定家園狀態 / 擁有少量指定世界特產。
- 主線阻擋提示須明確;支線阻擋可保留神秘感。

### 驗收
- 五種地點類型各自正確結算內容。
- 被擋格無論能量多少都不能翻;條件在他處變滿足後變可翻,翻時仍扣能量。
- 阻擋判定每次即時對當前狀態計算,不依賴儲存的解除旗標。

## 10. 事件系統(EventSystem)

所有事件共用同一套資料驅動架構;小事件、劇情事件、收束事件本質相同,差異只在內容長度與呈現。

### 規則
- 地圖地點只負責「可否翻、扣多少、解鎖後觸發哪個事件」;事件本體負責播圖文 / 對話、選項、發獎勵、寫日記、觸發後續效果。
- 事件內容是一串內容頁(插圖佔位 + 文字 / 旁白 / 對話 / 選項)。
- 可帶選項;**選項不做永久懲罰或主線錯過**,只影響當下對話 / 日記文字 / 少量獎勵 / 小彩蛋。
- 透過 `UINavigation` 開 overlay 播放,結束後**還原到 caller**(原本那張地圖那一格)。
- 可發獎勵(資源 / 家具 / 任務道具 / 超級任務次數 / 能量)與效果(開新圖 / 解鎖系統 / 設旗標)。
- **回看**:可從日記回看;回看**不重發獎勵**、顯示當時所選選項;第一版不可改選。

### 驗收
- 翻到事件格 → 開 overlay 播該事件內容頁;有選項時顯示並正確分支。
- 結束正確發 rewards、套 effects、標記完成、寫日記,關閉後還原 caller。
- 同一事件回看:不重發獎勵、顯示當時選項、不可改選。

## 11. 資源 + 資源點

### 資源分層
| 類型 | 例子 | 可購買 | 用途 |
|---|---|---|---|
| 基礎建材 | 木材、石頭、布料、晶石 | 可 | 家園修復、普通裝飾、中階升級 |
| 普通特殊材 | 金屬片、玻璃珠 | 少量 | 中階升級、主題裝飾 |
| 世界特產 | 大千結晶、魔法孢子 | 不直接賣 | 特定世界升級、事件修復、特殊阻擋 |
| 關鍵物 | 世界核心、故事信物 | 不可 | 主線、劇情、重大解鎖 |

- 每個平行世界 1 種世界特產(不綁單顆星球);同一世界不同區域決定基礎資源偏向。
- 可購買資源只能加速家園建設;世界特產與關鍵物主要來自探索,避免付費跳過地圖與故事。

### 資源點
- 每張圖**不一定**有資源點;有的話初次解鎖給首次發現資源(在翻開時結算,見 §8)。
- 解鎖後可投入能量重採(成本 / 每日次數見 §5);產出 = 世界特產 + 區域偏向基礎資源,不給能量、不給關鍵物。
- **集中採集頁**:列出所有已解鎖地圖上已開的資源點 + 今日可採狀態 / 成本 / 產出傾向;**第一版只做單點採集**(一鍵採多個延到 Phase 2)。

### 驗收
- 資源加 / 扣經 `GameState` API;扣到不足時回失敗不扣。
- 已開資源點依每日次數可重採;重採扣能量、發產出;次數用完當日不可再採,隔日恢復。
- 集中採集頁正確列出已開資源點與今日狀態;單點採集正確;每點各自顯示成本。

## 12. 家園系統(HomeSystem)

家園 = 主角的載具(露營車,固定基地),天生可跟著主角換星球。資源的主要出口、玩家長期累積感的載體。

### 規則
- **修復 / 升級**:消耗 Content 指定資源,推進修復等級,產生**視覺變化**(Phase 1~2 佔位,Phase 4 真美術)——成就感來自世界更完整,非數值更高。
- **插槽式佈置**:預定義固定位置 / 數量的插槽,每個插槽限制可放置的物件類型;玩家放置 / 替換 / 收納已擁家具。第一版**不做**任意拖放 / 無限堆疊 / 改房間格局。
- **家具只給視覺 / 收藏 / 自我表達,無任何能量或探索數值加成。** 可允許少量便利入口型家具(任務板 / 日記桌 / 商店櫃台),但不給數值收益。

### 驗收
- 修復只有持有指定資源時可行;修復精準扣資源、推進等級、改變(佔位)視覺;資源不足則不能修並給明確回饋。
- 插槽只接受其允許的家具類型;放置 / 替換 / 收納正確。
- 放置 / 收納家具完全不改變能量或探索數值。

---

## 13. 背包系統(檢視層)

主導航之一,是玩家探索、收藏、佈置與回憶的總入口;**不持有獨立狀態**,只分類顯示 `GameState` 既有資料 + 管理操作。

| 分類 | 內容 | 操作 |
|---|---|---|
| 資源 | 木 / 石 / 布 / 晶 / 世界特產 | 查看數量,用於建設 / 製作 |
| 任務道具 | 鑰匙 / 齒輪 / 信物等 | 查看用途與獲得方式,通常不手動使用(由 requirements 自動判定) |
| 家具 | 可放入插槽的物件 | 放置 / 替換 / 收納 |
| 外觀 | 載具與家具外觀 | 套用 / 更換(不給數值) |
| 收藏 | 事件插圖 / 紀念物 | 回看,不一定可消耗 |

### 驗收
- 各分類正確反映 `GameState` 對應資料;背包不自行持有與 `GameState` 不一致的狀態。
- 物品盡量記錄來源;任務道具不需手動使用即由 requirements 判定。

## 14. 日記系統(Diary)

自動記錄玩家現實行動與探索事件,讓「每天做的小事」與「世界變完整」可被回顧。

### 規則
- **每日自動紀錄**:日期 / 第 N 天、完成任務、獲得能量、探索格數、解鎖事件、家園升級、心情筆記(可空白)。資料由各系統「邊做邊記」(完成任務 / 翻格 / 修復當下寫入)。
- **事件收藏**:記錄第幾天觸發哪些事件;已解鎖事件可回看圖文(見 §10 回看規則)。
- **心情筆記**:當日所有任務完成後跳提示,可現在寫 / 稍後寫;之後仍可從日記補寫 / 編輯;**不影響獎勵 / 能量 / 進度**。

### 驗收
- 每日摘要正確累積(任務 / 能量 / 翻格數 / 事件 / 家園變化)。
- 已解鎖事件可從日記回看圖文。
- 當日全完成 → 跳心情筆記提示;當下不寫之後仍可補寫 / 編輯;心情筆記不動任何進度。

## 15. Content 資料模型

「怎麼走、怎麼吃」由資料定義的地方,也是內容生產的入口。

### 邊界:Content 定結構,全域設定定數字
- **Content**:有哪些地圖、格子怎麼連、哪格什麼類型、觸發哪個事件、事件給什麼、家園有哪些插槽 / 修復項——**關係與結構**;以及**逐格成本**與**單點劇情給予量**(某事件送 3 木材、某新手事件 +1 能量)。
- **全域設定(§5)**:跟內容無關的全域節奏數字(任務能量、上限、門檻、重採成本、每日次數、週超級門檻)。

### 各類資料
| 類 | 內容 | 組織 |
|---|---|---|
| 任務池 | 普通 / 高級兩組;每筆:id、類別、顯示文字 | 單一池 |
| 地圖 | id、名稱、平行世界、區域、資源偏向、起點格、含哪些格 | 一圖一檔 |
| 格子 | id、視覺位置、**鄰居清單(graph)**、**自帶成本**、requirements、內容類型 + 類型專屬欄位(事件 id / 資源量 / 家具 id / 出口圖 id) | 隨地圖 |
| 事件 | id、標題、規模標籤、發生地點、內容頁、選項、獎勵、效果、日記、可否回看 | **按區域 / 地圖分檔** |
| 物品 / 資源 metadata | 資源四層分類、任務道具(持有 / 消耗型)、家具(可放哪類插槽)、外觀;是否可堆疊 / 可購買 / 用途 | 集中 |
| 家園 | 插槽(id、位置、可接受類型)、修復項(id、需求資源、修復後視覺狀態) | 集中 |

### 驗收
- Content loader 讀資料成功,查詢介面回正確物件;壞資料報錯不崩。
- 格子成本來自格子自身資料;任務能量等全域數字來自全域設定。
- 事件按區域 / 地圖分檔可正確載入與查詢。

## 16. UINavigation(導航 + overlay 管理)

管「同時只開一個大頁面、overlay 疊上凍結底層、關閉還原 caller」的狀態機。**契約完整、實作薄**(Phase 1 純色佔位切換,無轉場 / skin;Phase 4 才加表現)。

### 規則
- **根頁互斥**:任務 / 地圖 / 家園 / 背包 / 日記 / 商店,同時只一個 active,底部導航切換。
- **overlay stack**:事件播放 / 商品詳情 / 確認對話疊上來 → 凍結底層輸入;關閉 → 還原到 caller 狀態 / 焦點。
- **toast / 紅點**走另一條(非阻擋),不進 overlay stack。

### 驗收
- 同時只一個根頁 active。
- 開 overlay 凍結底層輸入;關閉還原到 caller;事件 → 確認對話這種疊兩層也正確還原。
- toast / 紅點不阻擋互動。

---

## 17. AdService(介面)

封裝廣告 SDK 的抽象層,沿用 spike 驗證過的介面;遊戲邏輯只呼叫請求並監聽成功 / 失敗,使 SDK 可替換。

### 規則
- 對外只暴露:請求 rewarded、成功 signal(發獎勵)、失敗 signal(載入 / 播放失敗,帶原因)。
- 呼叫端不直接碰 plugin 類別。
- **優雅降級**:廣告載入失敗須有明確回饋或替代路徑,不可卡死玩家。
- Phase 1 為 **stub**(呼叫即成功,不播);Phase 3 換真 plugin,呼叫端不改。

### 驗收
- 呼叫 rewarded(stub)觸發成功 signal。
- 失敗時觸發失敗 signal 並帶原因;呼叫端能據以給回饋,不卡死。
- rewarded 廣告單元 ID 等常數只出現在 `AdService` 一處。

---

## 18. SaveService

第一版以本機存檔為主,不做自有帳號;玩家遊玩不需登入。

### 規則
- 存檔內容 = 序列化的 `GameState`(見 §4.4)。
- 提供存檔 / 讀檔 / round-trip。
- 日記與心情筆記只存本機,不上傳、不分析、不送 AI / 後端。
- 雲端 / 換機恢復非第一版必做(Phase 5 評估 iCloud / Play Saved Games)。
- 平台購買權益依 App Store / Google Play 帳號恢復,不等於恢復完整進度。

### 驗收
- 存檔後讀回 = 原狀態(含當日任務 / 日記 / 旗標)。
- 一次性資源包恢復購買只恢復「已購買」狀態,不重複發放資源。

## 19. 變現系統(架構;Phase 3 實作)

免費遊玩,收入來自 rewarded ad、一次性無廣告、外觀裝飾、家園資源包;**不直接賣能量 / 任務完成 / 主線探索進度**。架構於此確立,UI 與平台接入在 Phase 3。

| 商店 | 內容 |
|---|---|
| 遊戲商店 | 遊戲內資源購買、每日免費領取、rewarded ad 領取入口、圖紙製作商品 |
| 真錢商店 | 平台 IAP:無廣告、付費外觀 / 家具、一次性資源包、主題套裝;須支援恢復購買;第一版無可重複購買消耗品 |

- **無廣告**:一次性購買;購買後原需看廣告的獎勵改點擊直接領取,仍有每日次數上限;買的是免看廣告的便利,不是更多產出。
- **每日免費領取 / 每日資源補給**:小量基礎資源;不給能量 / 世界特產 / 關鍵物 / 主線解鎖物。
- **圖紙**:解鎖遊戲商店中的可製作商品(不做獨立製作頁);可來自商店 / 事件 / 探索 / 劇情。

### 驗收(架構面)
- 變現相關狀態(購買權益、每日領取旗標)在 `GameState`,可序列化恢復。
- rewarded 入口透過 `AdService`;無廣告權益正確改寫領取路徑。

## 20. Phase 規劃

> 各子階段的**實作契約**(檔案 / 簽名 / 接線)於該階段開工時寫入 `開發設計方針.md`;**手動驗收清單**於規格定稿時寫入 `測試指南.md`。本書只記範圍與**結束時驗收意圖**。

### Phase 1 — 純系統地基

目標:把所有機制以「可被呼叫的契約 + 最小真實行為」立起來,**全程 fixture、零真內容**。結束時 = 一台跑得動、接縫留好的機器;headless 邏輯測試全綠;1-B~1-E 各有 seam 可接。

| Sub | 內容 | 結束時驗收意圖 |
|---|---|---|
| **1-A 架構地基** | `GameState`(原子 + 查詢)、全域設定、Content 載入(地圖用相鄰 graph、格子自帶成本)、`SaveService`、`AdService` stub、`Event` seam、`requirements` 即時判定、`UINavigation` 契約(薄實作)、啟動 / debug 場景 | 新遊戲初始狀態正確;原子操作改值並發 signal;存檔 round-trip 無損;requirements 判定正確;Content loader 讀 fixture 成功、壞資料報錯不崩;AdService stub 觸發成功 signal;UINavigation 根頁互斥 + overlay 凍結 / 還原(含疊兩層)。**幾乎全 headless 可驗。** |
| **1-B 任務 + 能量** | TaskSystem(候選 / 槽位 / 常用 / 自我申報 / 超級任務 / 看廣告額外高級 stub)、能量產出規則(上限 / 跨日 / 輕提示)、每日重置接線、佔位任務頁 | §6 + §7 驗收;完成任務只給固定能量(不發新手贈點);advance_day 重置當日 / 保留結轉能量。**(此階段只「產」能量;「耗」在 1-C)** |
| **1-C 探索翻格** | Exploration(相鄰 graph / 霧 / 逐格成本 / 岔路)、五種地點類型結算、requirements 即時阻擋、多地圖切換、資源點**首採**、佔位地圖頁 | §8 + §9 驗收;到此核心循環機制上整條通(產 + 耗),可用 fixture 驗「翻得動 / 會卡 / 會見底」。 |
| **1-D 事件 + 日記** | Event 播放器(pages / choices / rewards / effects / diary,走 overlay + caller 還原)、completed_events 三用、日記頁 + 事件回看、心情筆記 | §10 + §14 驗收;日記資料由各系統邊做邊記。 |
| **1-E 家園 + 資源點** | HomeSystem(修復 → 佔位視覺、插槽擺放)、資源點重採 + 集中採集頁(單採) | §11(資源點)+ §12 驗收。 |

### Phase 2 — 內容生產 + 測試(薄原型模式)

目標:在 Phase 1 的機器上,用佔位畫面**做完並測完第一版全部內容**。

- **檢查點① — D1–D7 手感閘門**:做出約 3–4 張大千圖 + 1–2 張第二世界圖(夠跑完設計文件 D1–D7 流程),含真事件 / 劇情 / 新手贈點事件、真任務池、真家園 / 資源點定義;提供**跳日鈕**(dev-only)與 **headless 七日數值模擬**(dev-only)。
  - **驗收意圖**:D1–D7 真內容可玩到底;跳日正確;七日曲線合理;**主觀手感達標**——每日可翻 4–6 格、Day6 不斷崖、「翻完還想再翻 / 明天有東西等我」鉤子成立。(此為能量經濟 de-risk 閘門,過了才量產其餘內容。)
  - **誠實邊界**:跳日測節奏與機制手感;**真留存(D1/D7)只能靠跨真實天數玩近成品版**,不在此驗。
- **過閘門 → 完成第一版全部內容**:~20 圖、大千 3 區 + 第二世界 2 區、主線 + 部分支線、世界特產、出口鏈、阻擋流程、第二世界穿梭、家園完整修復 / 佈置、圖紙、背包完整分類。系統不動,只灌內容。

### Phase 3 — 變現整合
stub 換真 `AdService`、遊戲商店 + 真錢商店、無廣告、每日免費領取、rewarded 額外任務 / 資源補給、恢復購買、優雅降級。

### Phase 4 — 美術整合 + 正式 UI
ArtBible 換掉佔位:等角地圖、露營車內外、事件插圖、UI skin、動態回饋。系統照舊,只換表現層。

### Phase 5 — 平台 / 存檔 / 通知 / iOS
本機存檔落地、基礎通知與回訪提醒(只提醒不責備、每日最多 1 則合併)、iOS export + ATT、必要觸控化。

### Phase 6 — Endgame 雛形 + 平衡收尾 = 第一階段完整版
隨機地圖模板、照顧型回饋雛形、整體平衡、D1/D7 留存埋點。
