## [P2B-add-table-furniture-for-slot-desk] slot_desk 無對應 table 家具，永遠空著；需新增一件 table 家具（採選項 A） — ✅ 已修正

> 2026-06-04 進 2-C 前內容收尾。使用者決策：採**選項 A**（補一件 table，而非移除空 slot）。歸 **2-B 補正**，不進 2-C（2-C 不量產內容，加家具會污染手感閘門基準）。

### 問題

- `data/home.json` 的 `slot_desk`（`accepts: ["table"]`）在 data 裡**沒有任何 `furniture_type: table` 的家具**可放——三件家具（雨滴風鈴 / 晶谷風鈴 / 石丘燈籠）全是 `decor`。
- 結果：`slot_desk` 在家園頁永遠空著；且 decor 家具有 3 件、收 decor 的插槽只有 2 個（`slot_window_hook` / `slot_wall_left`），有一件 decor 無處可掛（附帶現象，本項先不處理）。

### 修改方向（實作者）

1. `data/items.json` 新增一件 `kind: furniture` / `furniture_type: table` 的家具（id、display_name、取得來源自訂；建議掛在 D1–D7 動線某張圖的 `decor` tile `reward_id` 或事件 rewards，讓玩家在閘門窗內能拿到並放上 slot_desk）。
2. 確認新增後 Content loader 不報錯、家具可成功放入 `slot_desk`。
3. 注意這是 2-B 收尾範圍的「最小補一件」；家園**完整**佈置（多桌面物 / decor 溢位調整）仍屬 2-D 量產，不在此項。

### 影響範圍

- 純內容缺口，不卡載入、不卡 D1–D7 動線；但會讓 2-C 軟手感「家園看得到成長」打折（留一個永遠空的插槽）。故須在 2-C 實玩前補。
